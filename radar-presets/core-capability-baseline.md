# Core Capability Baseline

These capability areas stay on for every `Signal Scout` user.

They are not optional presets. They are the default capability surface because they match the real outputs people want from `Codex` and `Claude Code`.

## Always-On Capability Areas

### 1. Build Websites

Track:

- frontend workflow skills
- browser QA and visual review
- preview and deploy loops
- design-to-code guidance

Adopt now:

- `react-best-practices`
- `playwright`
- `vercel-deploy`

Experiment:

- `web-design-guidelines`

### 2. Build Apps

Track:

- safe app scaffolds
- implementation discipline
- test loops
- packaging and local tool patterns

Adopt now:

- `test-driven-development`
- `systematic-debugging`
- `verification-before-completion`

Watch:

- CI repair workflows

### 3. Make Presentations

Track:

- slides and deck workflows
- research-to-deck systems
- story structure patterns
- visual communication tooling

Adopt now:

- `slides`

Experiment:

- `screenshot`
- spreadsheet-backed deck workflows

### 4. Research & Writing

Track:

- synthesis workflows
- source evaluation
- note-to-draft pipelines
- citation-aware writing

Adopt now:

- research-to-draft workflows

Experiment:

- citation-aware skill packs

### 5. Automate Work

Track:

- repeatable workflows
- safe MCP integrations
- connector ecosystems
- agent-assisted recurring tasks

Adopt now:

- `Context7`
- `Sequential Thinking`
- `GitHub MCP` read-only

Reject in core:

- write-capable MCP bundles by default

## Core Principle

Do not ask a user to choose between these five outputs.

The user already chose them by installing `Codex` or `Claude Code`.

Personalization starts only after this baseline.
