# Notes

Create one note per workstream or source cluster.

Recommended note shape:

- Source ID
- Source
- Date accessed
- Role: official / implementation / field / adversarial
- Platform: github / youtube / x / telegram / reddit / discord / forum / linkedin / facebook / other
- Artifact type: doc / repo / issue / video / thread / post / talk / catalog / other
- Option or candidate covered
- Entity type: standard / official-catalog / community-catalog / workflow-framework / translation-layer / other
- Maintainer type: vendor / company / community / individual / other
- Stars / forks / updated_at when relevant
- Claim tags
- Key points
- Numbers, dates, and versions
- What this supports
- What this contradicts
- Confidence / limitations
