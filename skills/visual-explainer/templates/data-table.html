What Victoria specified

Victoria's email (thread dated 2026-03-02, subject "Platform v2 final requirements") states explicitly: "All report views must support CSV and PDF export. PDF must match the printed layout used by finance."

What the implementation plan covers

The implementation plan includes export/csv.ts with full column mapping and a streaming response for large datasets. PDF export is not listed in any task, milestone, or file inventory.

Risk

  • Finance team uses PDF export for monthly board packs — this is a hard dependency, not optional.
  • Puppeteer or a headless renderer will be needed; this is non-trivial to add post-build.
  • Print-layout CSS must match the existing Word template Victoria attached.

Recommended action

Add a PDF export task to the implementation plan before sprint kickoff. Suggest export/pdf.ts using Puppeteer with a @media print stylesheet derived from the finance template. Estimate: 2–3 days. Assign to the same engineer handling CSV export for context continuity.
