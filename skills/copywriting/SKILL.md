# Copywriting

Use this skill when the task is:

- improve landing page copy
- sharpen a headline, subhead, or CTA
- tighten creator messaging, hooks, or titles
- improve business positioning or deck language
- turn rough notes into stronger user-facing copy

Do not use this skill for:

- generic proofreading only
- deep research or source synthesis
- implementation or code review work
- SEO-only metadata work with no messaging decision

## Trigger cases

- "Write better copy for this landing page."
- "Fix this CTA."
- "Make this creator page sound stronger."
- "Tighten the messaging in this deck."

## Non-trigger cases

- "Build the website."
- "Research the market."
- "Review this diff."
- "Fix spelling in this paragraph."

## Workflow

1. Lock the copy target first.

Write down:

- the asset type
- the audience
- the one action the reader should take next

2. Extract the actual offer.

Identify:

- what is being offered
- why it matters now
- what proof exists
- what the reader should understand in one sentence

If this is fuzzy, say so before rewriting.

3. Choose the copy shape.

Default patterns:

- landing page: headline -> subhead -> proof -> CTA
- creator page: hook -> promise -> proof -> next action
- deck section: claim -> evidence -> implication
- product section: problem -> outcome -> mechanism -> CTA

4. Write for clarity first.

Prefer:

- concrete outcomes
- short sentences
- plain language
- one idea per block

Avoid:

- vague hype
- stacked buzzwords
- multiple competing CTAs
- promises the product cannot support

5. Produce options where framing matters.

Give 2 to 3 options for:

- headlines
- CTAs
- value propositions
- titles

6. Check the copy against reality.

Verify:

- the copy matches the actual product or feature
- proof claims are supported
- the CTA is real
- tone fits the audience

7. End with a practical delivery.

Use:

- `Recommended copy`
- `Alternate options`
- `Why this is stronger`
- `Open gaps` if strategy or proof is still weak

## Review standard

A good copywriting pass should:

- clarify the offer
- reduce fluff
- improve scanability
- strengthen the next action

## Guardrails

- Do not invent proof, metrics, or testimonials.
- Do not hide a strategy problem behind prettier words.
- Do not confuse visual polish with messaging quality.
