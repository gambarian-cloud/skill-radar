# Illness Scripts

## Purpose

Illness scripts are structured mental representations of diseases stored in expert clinicians' long-term memory. They enable rapid pattern recognition (System 1 reasoning) while providing a framework for systematic comparison when the diagnosis is uncertain. This skill covers how to build, use, and teach illness scripts.

## The Illness Script Framework

Each illness script contains five core domains:

### 1. Epidemiology (Enabling Conditions)

The context that makes a disease possible or likely:

- **Demographics** — Age, sex, ethnicity, geographic location
- **Risk factors** — Behavioral (smoking, alcohol), genetic predisposition, occupational exposures
- **Predisposing conditions** — Immunosuppression, diabetes, pregnancy, recent surgery
- **Temporal context** — Season (influenza in winter), time since exposure, post-procedural interval
- **Setting** — Community-acquired vs nosocomial, endemic region, travel history

### 2. Pathophysiology (Fault)

The underlying mechanism driving the clinical presentation:

- **Primary process** — Infection, autoimmunity, malignancy, ischemia, obstruction
- **Anatomic location** — Organ, tissue, vascular territory
- **Pathologic mechanism** — Inflammation, necrosis, fibrosis, neoplastic proliferation
- **Systemic effects** — Mediator release, metabolic consequences, immune activation

Understanding pathophysiology enables prediction of expected findings and logical derivation of features even when the script is incomplete.

### 3. Time Course

The temporal evolution of the illness — often the single most discriminating feature:

- **Onset** — Hyperacute (seconds-minutes), acute (hours-days), subacute (days-weeks), chronic (weeks-months), insidious (months-years)
- **Progression** — Steadily worsening, relapsing-remitting, stepwise, self-limited, fluctuating
- **Duration** — Expected course if untreated, time to key complications
- **Response to treatment** — Expected timeline for improvement, red flags for non-response

Examples: Subarachnoid hemorrhage = thunderclap onset. Temporal arteritis = weeks of escalating headache. Tension headache = chronic, episodic.

### 4. Key Clinical Features

The findings that define the clinical presentation:

- **Cardinal symptoms** — The hallmark complaints (e.g., colicky flank pain radiating to groin in ureteric calculus)
- **Physical examination findings** — Specific signs (e.g., Murphy's sign in cholecystitis, Kussmaul breathing in DKA)
- **Laboratory features** — Expected abnormalities (e.g., elevated lipase >3× ULN in acute pancreatitis)
- **Imaging findings** — Characteristic appearances (e.g., ground-glass opacities in PJP pneumonia)
- **Negative findings** — Important absences that support the diagnosis (e.g., absence of fever in pulmonary embolism)

### 5. Complications

Expected consequences if the disease progresses or is untreated:

- **Early complications** — Occurring within the acute phase
- **Late complications** — Developing over weeks to months
- **Chronic sequelae** — Long-term consequences
- **Treatment-related complications** — Iatrogenic risks

## Building Illness Scripts

### For Learners

1. **Start with prototypical cases** — Learn the classic presentation first
2. **Add variants progressively** — Atypical presentations, age-specific differences, comorbidity effects
3. **Link to pathophysiology** — Each clinical feature should connect to an underlying mechanism
4. **Compare and contrast** — Build scripts in clusters of related conditions (e.g., all causes of acute abdominal pain)
5. **Use real cases** — Scripts built from patient encounters are more robust than those from textbooks

### Script Activation in Practice

When a clinician encounters a patient:

1. Early cues (chief complaint, demographics, vital signs) activate candidate scripts
2. Additional history and examination findings are compared against activated scripts
3. The script with the best "fit" rises to the top of the differential
4. Discrepant features prompt either script modification or script switching

## Pattern Recognition vs Analytical Reasoning

### Dual-Process Theory in Diagnosis

- **System 1 (Non-analytical)** — Fast, automatic, pattern-based. Uses illness scripts and pattern recognition. Dominant in experienced clinicians encountering familiar presentations.
- **System 2 (Analytical)** — Slow, deliberate, effortful. Uses systematic frameworks (VINDICATE, Bayesian reasoning). Dominant in learners and in unfamiliar presentations.

Expert clinicians use both systems, switching between them as needed.

### When Pattern Recognition Excels

- Typical presentations of common diseases
- Time-pressured situations (resuscitation, triage)
- Visual pattern recognition (dermatology, radiology, pathology)
- When the clinician has extensive experience with the condition

### When Analytical Reasoning Is Required

- Atypical or ambiguous presentations
- Unfamiliar conditions
- Complex multimorbid patients
- When the initial pattern match leads to a dead end
- Whenever System 1 generates a diagnosis that "doesn't feel right"

## Illness Script Comparison Table

When the differential is uncertain, construct a comparison table:

| Feature | Diagnosis A | Diagnosis B | Diagnosis C | This Patient |
|---|---|---|---|---|
| Epidemiology | ... | ... | ... | ... |
| Onset/Time course | ... | ... | ... | ... |
| Key symptom 1 | ... | ... | ... | ... |
| Key symptom 2 | ... | ... | ... | ... |
| Key lab finding | ... | ... | ... | ... |
| Key imaging | ... | ... | ... | ... |

This forces explicit comparison and highlights discriminating features — findings that are expected in one diagnosis but not another.

## Discriminating Features

The most powerful diagnostic clues are features with high likelihood ratios for one diagnosis and low likelihood ratios for competing diagnoses:

- Present in the target disease but absent in alternatives (pathognomonic or near-pathognomonic findings)
- Absent in the target disease but present in alternatives (ruling-out findings)
- Different quality in target disease vs alternatives (e.g., character of pain, temporal pattern)

### Examples of High-Value Discriminators

- **Rigidity** in abdominal pain → peritonitis (not seen in functional pain)
- **Thunderclap onset** of headache → SAH until proven otherwise
- **Migratory arthritis** → Rheumatic fever, gonococcal arthritis, reactive arthritis (not OA or gout)
- **Painless jaundice** → Malignant obstruction (vs painful jaundice in choledocholithiasis)

## Common Script Errors

1. **Incomplete scripts** — Missing key features leads to missed diagnoses. Remedy: structured learning using all five domains.
2. **Overly rigid scripts** — Expecting every patient to present classically. Remedy: deliberately learn atypical presentations.
3. **Outdated scripts** — Disease presentations evolve (e.g., partially treated meningitis, antibiotic-modified infections). Remedy: continuous learning and script updating.
4. **Single-disease scripts** — Failing to account for comorbidity and dual pathology. Remedy: practice applying scripts to complex cases.

## Applying This Skill

When assisting with clinical reasoning:

- Present information in illness script format when describing diseases
- Highlight discriminating features that distinguish between differential diagnoses
- Flag when a presentation matches or deviates from the expected script
- Explicitly state the time course, as it is often the most powerful discriminator
- Note where pattern recognition is reliable versus where analytical reasoning is needed
